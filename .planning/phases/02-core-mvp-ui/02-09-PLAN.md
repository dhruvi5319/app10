<objective>
Fix the session initialization failure caused by `VITE_API_BASE_URL=http://localhost:8000` in the frontend env files. When this absolute URL is set, `apiFetch` constructs absolute HTTP URLs (e.g. `http://localhost:8000/api/sessions`) which the browser blocks as mixed content or rejects as cross-origin when the app is served over HTTPS or from a non-localhost host. Setting `VITE_API_BASE_URL=` (empty) makes `apiFetch` build relative `/api` paths that are correctly routed through the Vite dev-server proxy to the backend.

Purpose: Unblock all 8 remaining UAT tests ŌĆö every test was skipped because Test 1 (session init) failed.
Output: Both `frontend/.env.example` and `frontend/.env` with `VITE_API_BASE_URL=` (empty).
</objective>

  <action>
The root cause (from UAT.md Gaps section):

`VITE_API_BASE_URL=http://localhost:8000` in `frontend/.env.example` and `frontend/.env` causes
`apiFetch` in `frontend/src/api/client.ts` to build absolute URLs:

```
const BASE_URL = import.meta.env.VITE_API_BASE_URL ?? '';
const url = `${BASE_URL}${path}`;
// ŌåÆ "http://localhost:8000/api/sessions"  ŌåÉ blocked as mixed-content/cross-origin
```

When `VITE_API_BASE_URL` is set to the empty string, `url` becomes just `"/api/sessions"` ŌĆö a
relative path that the Vite dev-server proxy (configured in `vite.config.ts`) correctly forwards
to `http://localhost:8000`.

**Changes required:**

1. **`frontend/.env.example`** ŌĆö Change line 1 from:
   ```
   VITE_API_BASE_URL=http://localhost:8000
   ```
   to:
   ```
   # Leave empty to use Vite proxy (recommended). Set to absolute URL only for standalone deploys.
   VITE_API_BASE_URL=
   ```

2. **`frontend/.env`** ŌĆö Change line 1 from:
   ```
   VITE_API_BASE_URL=http://localhost:8000
   ```
   to:
   ```
   VITE_API_BASE_URL=
   ```

Do NOT modify `frontend/src/api/client.ts` or `frontend/vite.config.ts` ŌĆö both are already
correct. The client correctly falls back to empty string via `?? ''`, and the Vite proxy is
already configured to forward `/api` to `http://localhost:8000`.
  </action>

  </verify>
